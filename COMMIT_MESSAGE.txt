feat(archivist): Build complete auto-documentation pipeline

The Archivist agent now has a fully functional auto-documentation
infrastructure that captures ecosystem state and generates historical records.

## New Scripts

### scripts/generate_snapshot.py
- Creates timestamped point-in-time snapshots of ecosystem state
- Compares to previous snapshot for delta tracking (new/changed/removed)
- Outputs both JSON (machine-readable) and Markdown (human-readable)
- Supports custom labels for milestone snapshots
- Tracks: skills, agents, tools, references, examples

### scripts/update_changelog.py
- Analyzes git history for .claude/skills/ and .claude/agents/ changes
- Auto-generates changelog entries in Keep a Changelog format
- Categorizes: Added, Changed, Fixed, Deprecated, Removed, Security
- Extracts entity names from file paths
- Merges with existing CHANGELOG.md
- Supports versioned releases

### scripts/generate_progress_report.py
- Generates weekly or monthly progress reports
- Analyzes snapshot series for trends
- Calculates growth rates and metrics
- Lists all changes in reporting period
- Includes structured sections for manual input

### scripts/run_initial_archive.sh
- One-command setup script
- Creates .claude/archive/ directory structure
- Runs ecosystem data generation
- Creates initial snapshot with "initial" label
- Creates initial CHANGELOG.md with founding entry

## Archive Structure

.claude/archive/
├── snapshots/          # Point-in-time ecosystem snapshots
├── changelogs/         # CHANGELOG.md and archived versions
├── blog-drafts/        # Auto-generated blog post drafts
└── progress-reports/   # Weekly and monthly reports
    ├── weekly/
    └── monthly/

## Snapshot Format

Each snapshot creates:
1. JSON file: Full ecosystem state + delta
2. Markdown file: Human-readable report with:
   - Summary statistics table
   - Changes since last snapshot
   - Ecosystem health metrics
   - Tool usage patterns

## Changelog Format

Follows Keep a Changelog standard with categories:
- Added: New skills/agents/capabilities
- Changed: Updates and improvements
- Fixed: Bug fixes
- Deprecated: Features marked for removal
- Removed: Deleted entities
- Security: Security-related changes

## Usage

# Initial setup (one time)
./scripts/run_initial_archive.sh

# Regular usage
python3 scripts/generate_snapshot.py                    # Create snapshot
python3 scripts/update_changelog.py                     # Update changelog
python3 scripts/generate_progress_report.py --weekly    # Weekly report
python3 scripts/generate_progress_report.py --monthly   # Monthly report

# Milestone snapshots
python3 scripts/generate_snapshot.py --label "v1.0-release"

# Changelog for recent commits
python3 scripts/update_changelog.py --since "1 week ago"
